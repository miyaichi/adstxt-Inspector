# サプライチェーンガイド - デジタル広告の透明性を理解する

## 広告サプライチェーンとは？

専門的な詳細に入る前に、広告サプライチェーンとは何かを理解しましょう。

デジタル広告では、コンテンツパブリッシャー（ウェブサイトやアプリの所有者）は広告を表示したいと考え、広告主はユーザーに広告を表示したいと考えています。しかし、この2つの当事者の間には、パブリッシャーと広告主を結びつける複雑な企業ネットワークが存在することが多いです。このネットワークが「広告サプライチェーン」と呼ばれています。

広告のための配送サービスのようなものと考えてください：
- **パブリッシャー**は広告が表示されるスペースを所有しています（空の棚がある店舗のようなもの）
- **広告主**は表示したい広告を持っています（商品のようなもの）
- **サプライチェーン**は適切な広告を適切なスペースに届けるのを助けるすべての企業（配送トラック、倉庫、流通センターのようなもの）

## SupplyChain Objectとは？

SupplyChain Object（よく「schain」と略される）は、IAB Tech Labによって開発された技術標準で、この複雑なシステムに透明性をもたらすためのものです。これは基本的に、広告が広告主からパブリッシャーに配信される過程に関わるすべての企業を追跡するデジタル記録です。

### なぜこれが重要なのか？

1. **透明性**：広告主が自分のお金がどこに行くのかを正確に知れる
2. **不正防止**：悪意のある業者が偽の、または無許可の広告スペースを販売することを困難にする
3. **品質管理**：正当な広告在庫ソースの検証を可能にする
4. **効率性**：不必要な仲介業者を特定し排除するのに役立つ

## サプライチェーンはどのように機能するか？

### 主要なプレーヤー

1. **パブリッシャー**：広告が表示されるウェブサイトやアプリの所有者
   - 訪問者を引き付けるコンテンツを作成する
   - 販売する広告スペース（在庫）を持っている

2. **SSP／エクスチェンジ**：Supply-Side Platform（供給側プラットフォーム）と広告取引所
   - パブリッシャーが広告在庫を販売するのを支援
   - パブリッシャーを複数の潜在的な買い手につなぐ
   - 例：Google Ad Manager、Xandr、Magnite

3. **DSP／広告主**：Demand-Side Platform（需要側プラットフォーム）と広告を購入するブランド
   - 広告主が効率的に広告スペースを購入するのを支援
   - 多くのパブリッシャーにわたって自動的に広告インプレッションを購入
   - 例：The Trade Desk、Google DV360、Amazon DSP

ウェブサイトやアプリを訪問すると：
1. パブリッシャーの広告スペースが利用可能になる
2. 広告リクエストがサプライチェーンを通じて送信される
3. 広告主がリアルタイムで広告表示の入札を行う
4. 落札した広告が表示される
5. お金が広告主からサプライチェーンを通じてパブリッシャーに流れる

## 主要なファイルを理解する

### Ads.txt（Authorized Digital Sellers：認可されたデジタル販売者）

これはパブリッシャーがウェブサイトに配置する単純なテキストファイルで、どの企業が彼らの広告在庫を販売する権限を持っているかを公に宣言するためのものです。

**例：**
```
exchange.com, 1234, DIRECT, ab123
adnetwork.com, 5678, RESELLER, cd456
```

これは以下を意味します：
- `exchange.com`（ID `1234`）はパブリッシャーの在庫を直接販売できる
- `adnetwork.com`（ID `5678`）はパブリッシャーの在庫を再販売できる

### Sellers.json

これはSSPやエクスチェンジが公開するファイルで、彼らが代表するすべての販売者（パブリッシャーや仲介業者）を識別するためのものです。

**例：**
```json
{
  "sellers": [
    {
      "seller_id": "1234",
      "name": "Example Publisher",
      "domain": "example.com",
      "seller_type": "PUBLISHER"
    }
  ]
}
```

これは以下を意味します：
- このエクスチェンジは `1234` と識別される販売者と連携している
- この販売者は "Example Publisher"（ドメイン "example.com"）である
- この販売者は実際のコンテンツパブリッシャー（再販業者ではない）

### SupplyChain Object

これは入札リクエストに含まれるデータ構造で、広告リクエストがサプライチェーンを通じて取る完全な経路を示します。

**例：**
```json
{
  "schain": {
    "ver": "1.0",
    "complete": 1,
    "nodes": [
      {
        "asi": "publisher-exchange.com",
        "sid": "1234",
        "hp": 1
      },
      {
        "asi": "dsp-partner.com",
        "sid": "5678",
        "hp": 1
      }
    ]
  }
}
```

これは以下を意味します：
- 広告リクエストは `publisher-exchange.com`（販売者ID `1234`）から始まった
- その後 `dsp-partner.com`（販売者ID `5678`）を通過した
- チェーンは完全（情報の欠落なし）

## パブリッシャーがするべきこと

1. **正確なAds.txtファイルの管理**
   - すべての認可された販売パートナーをリストする
   - 正確な関係（DIRECTまたはRESELLER）を指定する
   - パートナーシップが変更されたら迅速に更新する

2. **SupplyChain情報の検証**
   - サプライチェーンの透明性を適切に実装するパートナーと協力する
   - あなたの在庫がチェーン全体で適切に識別されていることを確認する

3. **定期的なモニタリング**
   - Ads.txtが最新であることを確認する
   - 未認可の販売者があなたの在庫を販売していないか監視する
   - サプライチェーンデータが正確であることを確認する

## SSP/エクスチェンジがするべきこと

1. **正確なSellers.jsonファイルの管理**
   - システム内のすべてのパブリッシャーとリセラーをリストする
   - 正確な識別情報を含める
   - パブリッシャーとの関係が変更されたら更新する

2. **SupplyChain対応の実装**
   - 入札リクエストにサプライチェーン情報を適切に渡す
   - 販売者IDがシステム間で一致することを確認する
   - Ads.txtとSellers.json情報間の接続を維持する

3. **一貫性の検証**
   - 販売者のAds.txtエントリが正確であることを確認する
   - 適切な識別のない不審な在庫をブロックする
   - クリーンな市場を維持する

## 透明なサプライチェーンの利点

デジタル広告エコシステムのすべての参加者にとって：

1. **不正の削減**：未許可の当事者が偽の在庫を販売することを困難にする
2. **より良いROI**：広告主は自分のお金がどこに行くのかを知れる
3. **信頼**：より信頼性の高いエコシステムを作り出す
4. **効率性**：不必要な仲介業者を特定し排除できる
5. **コンプライアンス**：透明性に関する業界標準の向上に対応する
